# Internationalisation (i18n)

## Overview

The SPA uses **@solid-primitives/i18n** for translations. The provider wraps the entire app, and any component can call `useI18n()` to access the translator.

## Using Translations in Components

```typescript
import { useI18n } from "@/i18n";

function MyComponent() {
  const { t, locale, setLocale } = useI18n();

  return <p>{t("nav.network")}</p>;
}
```

`t(key)` returns the translated string for the current locale. If the key is missing it logs a warning and returns the key itself — no crashes.

## Available Locales

Locales are registered in `src/i18n/locales/index.ts`:

```typescript
export const localeRegistry = {
  en: { dict: en, label: "English", flag: "🇬🇧" },
  hi: { dict: hi, label: "Hindi",   flag: "🇮🇳" },
};
```

The locale preference is persisted in localStorage under the key `"hz-locale"`. On mount the provider reads this value and switches to the saved locale.

## Adding a New Locale

1. Create `src/i18n/locales/<code>.ts` exporting `dict` of type `RawDictionary`.
2. Copy the `en` locale as a base and translate the strings.
3. Register it in `src/i18n/locales/index.ts`:

```typescript
import { dict as fr } from "./fr";

export const localeRegistry = {
  en: { dict: en, label: "English", flag: "🇬🇧" },
  fr: { dict: fr, label: "Français", flag: "🇫🇷" },
  // ...
};
```

TypeScript will enforce that every locale satisfies `RawDictionary`.

## Dictionary Structure

Dictionaries are flat-ish objects with dot-notation keys grouped by namespace. The English locale is split across files under `src/i18n/locales/en/`:

```
en/
├── index.ts    # merges all namespace files
├── nav.ts      # navigation labels
├── layout.ts   # layout / shell strings
├── ui.ts       # generic UI labels (buttons, status, etc.)
├── widgets.ts  # sidebar widget strings
└── tools.ts    # utility / settings strings
```

Example from `nav.ts`:
```typescript
export const nav = {
  network: "Network",
  hq: "HQ",
  settings: "Settings",
  // ...
};
```

Keys are accessed as `t("nav.network")`, `t("ui.loading")`, etc. The `@solid-primitives/i18n` flatten utility converts nested objects to dot-path keys at runtime.

## Reactive i18n in Module navItems

Nav item labels should use the reactive accessor form so they update when the locale changes:

```typescript
navItem: {
  label: () => useI18n().t("nav.myfeature"),
  // ...
}
```

Static string labels like `label: "My Feature"` will not update on locale switch.

## Template Interpolation

For strings with dynamic values, use template syntax supported by `@solid-primitives/i18n`:

```typescript
// Dictionary
{ "items.count": "{{count}} items" }

// Usage
t("items.count", { count: 42 })  // "42 items"
```

## Provider Location

`I18nProvider` wraps the entire app in `App.tsx`:

```tsx
<I18nProvider>
  <Router>
    {/* ... */}
  </Router>
</I18nProvider>
```

`useI18n()` throws if called outside the provider — it must be used inside a component tree rendered by the app.
