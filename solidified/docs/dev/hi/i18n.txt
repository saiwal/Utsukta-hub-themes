# अंतर्राष्ट्रीयकरण (i18n)

## अवलोकन

SPA अनुवाद के लिए **@solid-primitives/i18n** उपयोग करता है। provider पूरे ऐप को रैप करता है और कोई भी कॉम्पोनेंट `useI18n()` कॉल करके translator एक्सेस कर सकता है।

## कॉम्पोनेंट में अनुवाद उपयोग करना

```typescript
import { useI18n } from "@/i18n";

function MyComponent() {
  const { t, locale, setLocale } = useI18n();

  return <p>{t("nav.network")}</p>;
}
```

`t(key)` मौजूदा locale के लिए translated string रिटर्न करता है। अगर key नहीं मिली तो warning log होती है और key खुद रिटर्न होती है — कोई crash नहीं।

## उपलब्ध Locale

Locale `src/i18n/locales/index.ts` में रजिस्टर होते हैं:

```typescript
export const localeRegistry = {
  en: { dict: en, label: "English", flag: "🇬🇧" },
  hi: { dict: hi, label: "Hindi",   flag: "🇮🇳" },
};
```

Locale प्रेफ़रेन्स localStorage में `"hz-locale"` key के तहत सेव रहती है। माउंट होने पर provider इस value को पढ़कर saved locale पर switch करता है।

## नया Locale जोड़ना

1. `src/i18n/locales/<code>.ts` बनाएं जो `RawDictionary` type का `dict` export करे।
2. `en` locale की कॉपी base की तरह लें और strings translate करें।
3. `src/i18n/locales/index.ts` में रजिस्टर करें:

```typescript
import { dict as fr } from "./fr";

export const localeRegistry = {
  en: { dict: en, label: "English", flag: "🇬🇧" },
  fr: { dict: fr, label: "Français", flag: "🇫🇷" },
  // ...
};
```

TypeScript सुनिश्चित करेगा कि हर locale `RawDictionary` को satisfy करे।

## Dictionary संरचना

Dictionary dot-notation keys वाले flat-ish objects हैं जो namespace के अनुसार ग्रुप हैं। English locale `src/i18n/locales/en/` के अंदर फ़ाइलों में बंटी है:

```
en/
├── index.ts    # सभी namespace फ़ाइलें merge करता है
├── nav.ts      # नेविगेशन लेबल
├── layout.ts   # layout / shell strings
├── ui.ts       # generic UI लेबल (बटन, status आदि)
├── widgets.ts  # sidebar widget strings
└── tools.ts    # utility / settings strings
```

`nav.ts` से उदाहरण:
```typescript
export const nav = {
  network: "Network",
  hq: "HQ",
  settings: "Settings",
  // ...
};
```

Keys को `t("nav.network")`, `t("ui.loading")` आदि के रूप में access किया जाता है।

## मॉड्यूल navItems में Reactive i18n

Nav item labels को reactive accessor form में रखें ताकि locale बदलने पर वे update हों:

```typescript
navItem: {
  label: () => useI18n().t("nav.myfeature"),
  // ...
}
```

`label: "My Feature"` जैसे static string लेबल locale switch होने पर update नहीं होंगे।

## Template Interpolation

Dynamic values वाली strings के लिए `@solid-primitives/i18n` का template syntax उपयोग करें:

```typescript
// Dictionary में
{ "items.count": "{{count}} आइटम" }

// उपयोग
t("items.count", { count: 42 })  // "42 आइटम"
```

## Provider की स्थिति

`I18nProvider` `App.tsx` में पूरे ऐप को रैप करता है:

```tsx
<I18nProvider>
  <Router>
    {/* ... */}
  </Router>
</I18nProvider>
```

`useI18n()` provider के बाहर call करने पर error थ्रो होती है — इसे ऐप के कॉम्पोनेंट tree के अंदर ही उपयोग करें।
