# System Logs

The Logs section shows the Hubzilla system log — a record of errors, warnings, and informational messages generated by the server.

[IMAGE: Logs section with filter bar and log entries]

## Log Entry Anatomy

Each log entry shows:
- **Timestamp** — when it was logged (hover for full precision)
- **Level badge** — severity classification
- **Message** — the log message text

[IMAGE: Individual log entry with timestamp, level badge, and message]

## Severity Levels

Entries are colour-coded by severity:

| Level | Colour | Meaning |
|-------|--------|---------|
| EMERG, ALERT, CRIT, ERROR | Red | Serious errors requiring attention |
| WARN | Amber | Warnings — things that may become problems |
| NOTICE, INFO | Blue | Informational messages — normal operation |
| DEBUG | Grey (dimmed) | Verbose diagnostic output |

## Filtering by Level

Use the filter buttons at the top of the log list to show only entries of a given severity:
- **All** — show everything
- **Error** — EMERG + ALERT + CRIT + ERROR only
- **Warning** — WARN only
- **Info** — NOTICE + INFO only
- **Debug** — DEBUG only

[IMAGE: Level filter buttons with Error selected]

**Recommended workflow:**
1. Start with **Error** filter — these always need a response.
2. Switch to **Warning** to find issues that might escalate.
3. Use **Debug** only when diagnosing a specific problem — it is very verbose.

## Common Error Patterns

| Pattern | Likely cause |
|---------|-------------|
| `queue` / `delivery` errors | Federation delivery failures — check remote server reachability |
| `database` errors | Database connection or query problems |
| `plugin` / `addon` errors | A misbehaving plugin — disable it and check if errors stop |
| `photo` / `file` errors | Storage permissions or disk space issues |
| `curl` / `network` errors | The server cannot reach remote hosts — check firewall / DNS |

## Refreshing

The log viewer loads on page open. Click **Refresh** (if available) or reload the page to see the latest entries.

[IMAGE: Refresh button on logs page]

> **Tip:** Set up external log monitoring (e.g. fail2ban, log aggregator) on your server for persistent alerting — the in-app log viewer is for spot-checks, not continuous monitoring.
