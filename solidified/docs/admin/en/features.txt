# Features

The Features section lets you enable or disable **optional site features** — things that are available in Hubzilla but not turned on by default. Changes affect all users on your hub.

[IMAGE: Features section showing grouped toggles]

## How It Works

Features are grouped into **sections** (e.g. "Post and Content", "Profile", "Network"). Each feature is a toggle with a label and brief description.

- **Checked** = feature is enabled for all users
- **Unchecked** = feature is disabled hub-wide

Click **Save** at the top right to apply your changes.

[IMAGE: Feature toggle rows with labels and checkboxes]

## Common Feature Groups

### Post and Content
Controls what users can do when writing posts — e.g. enabling spoiler tags, content warnings, custom permissions on individual posts, or additional formatting options.

### Profile
Controls what profile fields and features are available — e.g. enabling birthday display, keywords, or custom profile fields.

### Network / Stream
Controls stream-related features — e.g. advanced sorting options, the "following" filter, or specific view modes.

### Connection / Privacy
Controls privacy-related features available to users — e.g. enabling object capabilities, group-actor channels, or advanced permission interfaces.

## Impact

Feature changes take effect immediately after saving. Users do not need to reload — the next page load will reflect the new state.

> **Tip:** Enable features incrementally. Turning everything on at once can be confusing for users. Start with the features most relevant to your community and add more as users get comfortable.

## Saving

Click **Save** at the top right. A brief success message confirms the save. If an error occurs, an error message is shown and no changes are applied.
